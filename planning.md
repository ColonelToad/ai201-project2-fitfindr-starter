### Tool 1: search_listings

**What it does:**
Searches the dataset for items that match the user's text description, fit within a maximum price, and match a specific size string.

**Input parameters:**
- `description` (str): Keywords to look for in the listing's title, description, or style_tags.
- `size` (str): The specific size constraint (e.g., "M", "W30 L30"). Optional.
- `max_price` (float): The maximum allowed price for the item. Optional.

**What it returns:**
A list of dictionaries, where each dictionary represents a complete listing from listings.json that matches the criteria, sorted by price (lowest first).

**What happens if it fails or returns nothing:**
If no matches are found, it returns an empty list []. The planning loop will catch this and trigger a Smart Fallback (retry without the size constraint).

---

### Tool 2: suggest_outfit

**What it does:**
Analyzes the newly found item against the user's existing wardrobe, using an LLM prompt to evaluate color harmony and style tag overlaps to build a cohesive outfit.

**Input parameters:**
- `new_item` (dict): The single listing dictionary selected from search_listings.
- `wardrobe` (dict): The complete JSON object representing the user's closet (from wardrobe_schema.json).

**What it returns:**
A string containing a styling recommendation that explicitly names 1-2 items from the user's wardrobe and explains why the colors or proportions work well together.

**What happens if it fails or returns nothing:**
If the user's wardrobe array is empty, the tool will return a generic styling suggestion utilizing universal basics (e.g., "Pair this with classic baggy denim and chunky sneakers") and kindly remind the user to update their wardrobe data.

---

### Tool 3: create_fit_card

**What it does:**
Condenses the selected item and the outfit suggestion into a punchy, casual, Instagram-style text caption ready for social media.

**Input parameters:**
- `outfit_suggestion` (str): The styling text generated by suggest_outfit.
- `net_item` (dict): The dictionary containing the item's details (needed for price, brand, and platform context).

**What it returns:**
A short string formatted like a social media caption (lowercase aesthetic, emojis, mentioning the price and platform).

**What happens if it fails or returns nothing:**
If the outfit suggestion is somehow missing or corrupted, it falls back to generating a simple hype caption about the item itself: "just snagged this [title] off [platform] for $[price] ✨".

## Planning Loop

**How does your agent decide which tool to call next?**

The loop starts by extracting the search constraints from the user's natural language query and calls search_listings.

Conditional Branch 1 (Smart Fallback): It checks the length of the returned list. If len(results) == 0, it calls search_listings again, this time passing size=None to loosen the constraints.

Conditional Branch 2 (Hard Fail): If the results are still empty after the fallback, the loop sets an error message in the session and terminates early, returning to the user.

Main Flow: If results exist, it selects the first item (results[0]) and stores it in state.

It then sequentially passes that item and the loaded wardrobe into suggest_outfit, stores that result, and finally passes both into create_fit_card, stopping once the final string is generated.

---

## State Management

**How does information from one tool get passed to the next?**

The agent utilizes a central session_state dictionary initialized at the start of the run. It tracks:

user_query (str)

wardrobe_data (dict)

selected_item (dict)

outfit_idea (str)

final_fit_card (str)

After a tool executes successfully, the planning loop writes the return value to the corresponding key in session_state. Subsequent tools read directly from session_state for their inputs, eliminating the need for the user to repeat context.

## Architecture

```flowchart TD
    User([User Query]) --> Init[Initialize Session State]
    Init --> Extract[Extract Search Params]
    Extract --> Search1[search_listings]
    
    Search1 -- results == [] --> Fallback[Retry: search_listings w/o size]
    Fallback -- results == [] --> ErrorOut([Return: Tell user to adjust terms])
    
    Search1 -- results found --> StoreItem
    Fallback -- results found --> StoreItem
    
    StoreItem[State: selected_item = results[0]] --> Suggest[suggest_outfit]
    
    Suggest -- wardrobe empty --> GenericStyle[Generate Generic Outfit]
    Suggest -- success --> StoreOutfit
    GenericStyle --> StoreOutfit
    
    StoreOutfit[State: outfit_idea = result] --> FitCard[create_fit_card]
    
    FitCard --> StoreCard[State: final_fit_card = result]
    StoreCard --> Output([Return Final Response to User])
```

## A Complete Interaction (Step by Step)

Write out what a full user interaction looks like from start to finish — tool call by tool call. Use a specific example query.

**Example user query:** "I'm looking for a vintage graphic tee under $30. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?"

Step 1:
The planning loop extracts parameters and calls search_listings(description="vintage graphic tee", max_price=30.0, size=None). It returns a list of matches. The loop takes the first result (e.g., "Faded Band Tee — $22") and saves it to session_state['selected_item'].

Step 2:
The loop calls suggest_outfit(new_item=session_state['selected_item'], wardrobe=session_state['wardrobe_data']). The LLM evaluates the tee against the wardrobe and returns: "Pair this faded grey band tee with your 'Baggy straight-leg jeans, dark wash' and 'Chunky white sneakers' for a classic grunge silhouette." This string is saved to session_state['outfit_idea'].

Step 3:
The loop calls create_fit_card(outfit_suggestion=session_state['outfit_idea'], new_item=session_state['selected_item']). The LLM formats this into a shareable caption: "thrifted this vintage band tee for $22 and it pairs flawlessly with my dark wash baggies and chunky sneakers ✨ full fit in stories". This is saved to session_state['final_fit_card'].

Final output to user:
The agent prints out a cohesive response:
"I found a great option: Faded Band Tee for $22 on Depop.
How to style it: Pair this faded grey band tee with your 'Baggy straight-leg jeans, dark wash' and 'Chunky white sneakers' for a classic grunge silhouette.
Your Fit Card: thrifted this vintage band tee for $22 and it pairs flawlessly with my dark wash baggies and chunky sneakers ✨ full fit in stories"
